# Default PII / sensitive-content patterns for oh-my-codex-research.
# Each non-blank, non-comment line is an extended regex (POSIX ERE).
# pii-scrub.sh blocks any Write|Edit whose staged content matches.
#
# Override per-project by creating .omx/omxr/scrub-patterns.txt at the project
# root — that file wins entirely (defaults are NOT merged).
#
# Tune carefully: a too-strict pattern blocks legitimate writes;
# a too-loose pattern lets PII through. Test with `bash hooks/pii-scrub.sh`
# fed a sample JSON payload before committing changes.

# Email addresses
[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}

# Apparent US Social Security Numbers (strict 9-digit XXX-XX-XXXX form)
\b[0-9]{3}-[0-9]{2}-[0-9]{4}\b

# Generic 6-digit "subject ID" — common in HCP-style datasets. False-positive
# risk is real (any 6-digit number triggers); refine in .omx/omxr/scrub-patterns.txt
# if your project legitimately writes 6-digit numbers.
\bsubject[-_ ]?[0-9]{6}\b
